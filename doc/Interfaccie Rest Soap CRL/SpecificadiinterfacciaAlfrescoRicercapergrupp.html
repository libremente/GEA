 Executive Summary

Questo documento descrive tutte le interfacce esposte da Alfresco relative al dominio di interesse CRL.

Interfacce disponibili

Questo capitolo descrive alcune delle interfacce di Alfresco utilizzabili per poterlo integrare in diversi contesti Enterprise.

CMIS

Alfresco supporta l'interfaccia Content Management Interoperability Services (CMIS) in entrambi i bindings REST e SOAP.

Questo è il nuovo standard sviluppato appositamente per tutti i repository che consente il reale interfacciamento e riuso di implementazioni di interfacce per applicazioni sia repository-to-repository (R2R) che application-to-repository (A2R).

REST API

Alfresco espone un'interfaccia REST di default che contiene più di 300 metodi utilizzabili out-of-the-box.

Inoltre è possibile estendere la sua REST API tramite il framework Spring WebScripts per fare in modo di fare economia di richieste HTTP accorpando l'esecuzione di logiche di business custom in un'unica chiamata HTTP.

SOAP API

Questa è la prima API remote che storicamente è stata implementata in Alfresco. La SOAP API di Alfresco è basata su un linguaggio creato appositamente per eseguire operazioni multiple in un unico messaggio SOAP. La risposta verrà incapsulata in un unico messaggio contenente tutti gli esiti relativi alle operazioni processate dal repository. Questa API è espressamente orientata a scenari tipicamente di carattere di Enterprise Service Bus (ESB) in cui è necessario processare vettori di operazioni per la manipolazione dei contenuti permettendo una politica di economia di messaggi scambiati tra i sistemi.

RESTful API del dominio CRL

Tutte le funzionalità dell’applicazione web di front-end saranno fruibili tramite una specifica RESTful API esposta da Alfresco che sarà adattata al dominio di interesse della Gestione Atti CRL.

Di seguito sono descritti tutti i metodi che saranno esposti dall’applicazione di back-end Alfresco.

La REST API di dominio CRL sarà categorizzata all’interno di Alfresco secondo i seguenti criteri di Spring WebScripts:

family: CRL - Gestione Atti

package: crl/gestioneatti

Presa in carico

POST crl/atto/presaincarico

Parametri della richiesta:

  • id: id dell’atto
  • stato: stato da aggiornare sull’atto
  • dataPresaInCarico

Esempio di messaggio JSON per la richiesta:

{

            "atto" : {

                    "id" : "workspace://SpacesStore/8aba9d03-7293-4b75-a6ee-8c578cd63861",

                    "stato" : "bloccato in commissione",

                    "dataPresaInCarico" : "2012-09-10"

                    }

            }

Risposta:

  • 200 OK
  • 500 KO

Dettaglio atto

Sistemi esterni

In questo capitolo sono descritti alcuni scenari di integrazione di Alfresco con i sistemi esterni coinvolti.

Questo capitolo sarà completato solo a fronte delle necessarie informazioni riguardo tutti i sistemi impattati da Alfresco.

Si richiede, per ogni sistema che deve dialogare con Alfresco, la disponibilità di un documento di specifica di interfaccia per ogni sistema coinvolto.

Archiflow

L'interfacciamento per Archiflow verrà implementato tramite una web application separata che farà da tramite per gli infasamenti del protocollo tra CRL ed il repository Alfresco.

Anagrafica CRL

La modalità di interazione è basata sull'invio di file XML attraverso una delle API esposte da Alfresco. Il file XML appena salvato all'interno del repository sarà processato da una regola attiva nello spazio della drop zone che farà in modo di manipolare il file tramite una custom action che eseguire le dovute operazioni.
